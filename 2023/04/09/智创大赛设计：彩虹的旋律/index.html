智创大赛设计：彩虹的旋律

一、项目介绍

作品集三种功能于一体，既能通过手动演奏钢琴模块发声，又能读取屏幕上显示的色块合成乐音，同时，还可以在读取屏幕色彩变化的同时，亮起钢琴模块上相应的指示灯，以辅助用户演奏。

二、项目组成

三、项目功能介绍

弹奏模式

由用户进行弹奏，并根据不同的音符呈现不同的光彩。

自动播放模式

通过乐曲识别模块读取印有不同颜色的滚动纸带，或者滚动播放不同颜色的手机画面，将颜色转化为音符，并通过蜂鸣器播放出来。

指导弹奏模式

同“自动播放模式”一样，需要识别乐曲，但并不是直接播放，而是亮起不同的 LED 指示灯，指导用户进行弹奏。

带电开关机器

同时按下三个按钮，可以触发关机，所有灯光、蜂鸣器与触摸板等将失去作用。再次同时按下三个按钮，可以自动开机。

元器件型号

  • 单片机：Arduino UNO
  • 色彩识别模块：TCS34725
  • 钢琴触摸模块：TTP229 触摸IC
  • 蜂鸣器模块：Keyes 5V
  • 供电：9V DC 锂电池

程序设计

采用 C++ 作为 Arduino 编程语言，编程器选用 AVRISP mkll，Arduino IDE 版本为 1.8.19。

使用第三方库

  • FastLED：版本 3.5.0，用于钢琴背光 LED 的控制
  • MH_TCS34725：版本 1.0，用于色彩识别模块的驱动
  • TTP229：版本未知（已修改），用于钢琴触摸模块的驱动

代码组织结构

  • RainbowPiano.ino：程序入口，调度整个流程
  • ModuleStart.h：通过宏定义控制各个模块的启动，方便调试，见后文所示
  • PinHandler.h PinHandler.cpp：负责处理引脚初始化
  • TcsHandler.h TcsHandler.cpp：负责处理色彩识别
  • PianoHandler.h PianoHandler.cpp：负责处理钢琴触摸信号以及指示 LED 及背光 LED 的控制
  • BuzzerHandler.h BuzzerHandler.cpp：负责蜂鸣器的控制
  • Utils.h Utils.cpp：包含一些工具函数，如 RGB 色值转换、按钮状态读取、带电关机处理等
  • libMH_TCS34725.h libMH_TCS34725.cpp：第三方库，读取色彩识别信号
  • libTTP229.h libTTP229.cpp：修改过的第三方库，负责处理钢琴触摸

解决钢琴触摸阻塞的问题

事实上，官方库提供了两种读取方式：ReadKeys16() 与 GetKeys16()，前者会调用 WaitForTouch() 函数阻塞程序，而后者直接采用 IsTouch() 直接判断是否被触摸。但这两种方式均不满足预期的期望：前者会阻塞对按钮的处理与模式的切换，后者会干扰钢琴的识别，造成遗漏。

在整个过程中，我们还产生了另外两个思路：类多线程与中断函数。前者对程序架构影响较大，且操作复杂，而后者只有两个引脚适用（我们有三个按钮，虽然可以通过一些技巧让两个引脚获取三个按钮的状态），同时，在执行中断函数后，依然回到原位置继续进行等待。因此经过比较，上述解决方案最为优雅适用。

五、辅助程序设计

辅助程序使用 Python 语言设计，采用 PyQt6 绘制图形化界面，使用 opencv 合成视频，并自创了 mnote 格式文件，用以记录乐谱。该辅助程序开源于 Github：RainbowPiano。

该程序直接对标项目的各种功能，提供了键盘演奏、自动播放的功能，并为了方便用户使用，可以将音乐导出为视频（当然是通过颜色进行可视化）。
